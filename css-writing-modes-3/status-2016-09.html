CSS Writing Modes Level 3 Test Status 2016-09

This document analyzes tests that do not meet the exit criteria and are not required as of 2016.

At Risk Features

  1. Only 1 implementation for sideways-lr and sideways-rl.
  2. No unprefixed implementations form text-combine-upright: digits (1 prefixed.)
  3. No implementations for the look-ahead/look-behind sequencing rules for the text-combine-upright property.
  4. No implementations for the automatic triggering of multi-column behavior of orthogonal flows.

Tests That Require Other Specs

  1. No implementations for bidi control characters in UAX#9.
    • bidi-003
    • bidi-004
  2. Only 1 implementation that require Flexbox row-reverse
    • css-flexbox-row-reverse
  3. No implementations that require CSS Regions:
    • autoheight-regions-in-autoheight-flexbox-002
    • extract-ordered-list-direction-and-writing-modes
  4. Only 1 implementation for CSSOM View Range.getBoundingClientRect() method and loading a special web font.
    • text-orientation-script-001
  5. Only 1 implementation that require text-transform: full-width.
    • text-combine-upright-compression-005a

May/Should Features (Not Required To Exit)

  1. Only 1 implementation for form controls.
    • form-controls-vlr-004
    • form-controls-vlr-005
    • form-controls-vrl-004
    • form-controls-vrl-005
  2. Only 1 partial implementation for full-width characters to be converted to non-full-width characters for the text-combine-upright property.
    • full-width-001
    • full-width-002
    • full-width-003
    • text-combine-upright-compression-005a
    • text-combine-upright-compression-006

Required Tests That Do Not Meet Exit Criteria

  1. 4.3. Atomic Inline Baseline: These tests check an inline-block with block descendants aligns its synthesized baseline to the baseline of the linebox.

    No implementations pass these tests.

    • inline-block-alignment-003
    • inline-block-alignment-005
    • inline-block-alignment-007
    • inline-table-alignment-003
    • inline-table-alignment-005
  2. 5.1. Orienting Text: the text-orientation property: These tests are for text-orientation: upright defines: This value causes the used value of direction to be ltr, and for the purposes of bidi reordering, causes all characters to be treated as strong LTR.

    No implementations pass these tests.

    • table-progression-vlr-003
    • table-progression-vlr-004
    • table-progression-vrl-003
    • table-progression-vrl-004
  3. 7.1: Principles of Layout in Vertical Writing Modes: 7 among the 256 tests for absolute-positioned objects fail. These tests cover combinations of a) direction, b) text-align, c) box offsets (top, left, right, bottom, auto), d) orthogonal and non-orthogonal, and e) ICB.

    Only 1 implementation passes these tests.

    • abs-pos-non-replaced-vrl-088
    • abs-pos-non-replaced-vrl-220
    • abs-pos-non-replaced-vrl-224
  4. 9. Glyph Composition: Tests when the text inside text-combine-upright has a new line character. A similar test that has a space character has 2 implementations.

    Only 1 implementation passes these tests.

    • text-combine-upright-break-inside-001a
